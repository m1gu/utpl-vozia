# Plan: Normalización de Productos para Vapi

**Fecha**: 2026-06-26
**Objetivo**: Normalizar el campo `nombre_producto_interes` que llega desde CRM antes de enviarlo al asistente Vapi. Los valores crudos vienen con abreviaciones, typos, códigos internos y nombres inconsistentes. Este plan mapea cada variante a un nombre canónico y permite al asistente Vapi manejar casos sin documento asociado con un diálogo genérico.

---

## 1. Nueva tabla en Supabase: `utpl_productos`

```sql
CREATE TABLE public.utpl_productos (
  id SERIAL PRIMARY KEY,
  variante TEXT NOT NULL UNIQUE,
  nombre_canonico TEXT NOT NULL,
  tiene_documento BOOLEAN DEFAULT true,
  modalidad TEXT
);

COMMENT ON TABLE public.utpl_productos IS 'Catálogo de normalización de nombre_producto_interes para Vapi';
```

### Columnas

| Columna | Ejemplo | Descripción |
|---------|---------|-------------|
| `variante` | `ING RECUR NATURALES Y SOSTENIB` | Valor crudo tal cual llega del CRM |
| `nombre_canonico` | `Ingeniería en Recursos Naturales y Sostenibilidad` | Nombre limpio que recibe el asistente Vapi |

## 2. Nuevo nodo Postgres: `NORMALIZA PRODUCTO`

### Ubicación

Insertado entre `GUARDA PRE-LLAMADA` (output 0) y `PREPARA LLAMADA VAPI`, solo en el path que deriva en llamada Vapi.

```
Antes:
  GUARDA PRE-LLAMADA (out 0) → PREPARA LLAMADA VAPI → LLAMAR VAPI AGENTE

Después:
  GUARDA PRE-LLAMADA (out 0) → NORMALIZA PRODUCTO → PREPARA LLAMADA VAPI → LLAMAR VAPI AGENTE
```

El path de error (`GUARDA PRE-LLAMADA` out 1 → `FALLBACK RESTORE`) no se modifica.

### Parámetros

```json
{
  "parameters": {
    "operation": "executeQuery",
    "query": "SELECT nombre_canonico, tiene_documento\nFROM public.utpl_productos\nWHERE variante = '{{ $json.producto }}'\nLIMIT 1;",
    "options": {}
  },
  "type": "n8n-nodes-base.postgres",
  "typeVersion": 2.5,
  "position": [-4020, 980],
  "name": "NORMALIZA PRODUCTO",
  "credentials": {
    "postgres": {
      "id": "Iz7EEp0WgJu3LXhn",
      "name": "Gea_postgress"
    }
  }
}
```

### Conexiones nuevas

**ID del nodo**: `6d497c3b-9648-4223-a973-8ef401f38a9a`

**Agregar al final del `jsCode`**, después del bloque `// 4. Seleccionar según telefono_index`:

```javascript
// 5. Normalización de producto desde catálogo utpl_productos
const prodItems = $items('NORMALIZA PRODUCTO', 0, 0);
const prodRow = prodItems.length > 0 ? prodItems[0].json : {};
lead.producto_canonico = String(prodRow.nombre_canonico || lead.producto || '').trim();
lead.tiene_producto_especifico = prodRow.tiene_documento !== false;

// Fallback: si no hay match en el catálogo, usar el producto original
if (!lead.producto_canonico) {
  lead.producto_canonico = lead.producto;
  lead.tiene_producto_especifico = true;
}
```

### Variables nuevas que recibe cada asistente

### Texto a agregar en el system prompt de los 3 asistentes

```
## Comportamiento según tipo de producto

Evalúa la variable `tiene_producto_especifico`:

- Si `tiene_producto_especifico = true`: 
  Usa el valor de `nombre_producto` como el programa de interés del cliente. 
  Preséntalo con entusiasmo, menciona sus beneficios, duración, costo y campos de acción. 
  Responde preguntas específicas usando la herramienta de búsqueda en la base de conocimiento.

- Si `tiene_producto_especifico = false` o `nombre_producto` es "Consulta General":
  Realiza una presentación genérica de la UTPL. Saluda cordialmente, preséntate como 
  asesor virtual de la universidad, y pregunta al cliente qué tipo de carrera, 
  tecnología o posgrado le interesa para poder orientarlo. 
  No intentes hablar de un producto específico. 
  Si el cliente menciona un interés concreto, usa la herramienta de búsqueda 
  para encontrar información sobre ese tema.

En ambos casos, la variable `nombre_producto_original` contiene el nombre tal cual 
lo registró el cliente en el formulario. Puedes usarlo como referencia si el 
cliente lo menciona textualmente.
```

## 8. Notas adicionales

- La normalización es **no bloqueante**: si no hay match en `utpl_productos`, se usa el valor original del CRM como fallback.
- Los asistentes Vapi ya filtran por modalidad al consultar el RAG (`UTPL_RAG_buscar.json`), por lo que nombres iguales en Mel y Posgrado (ej: "Pedagogía de los Idiomas Nacionales y Extranjeros") no generan ambigüedad.
- La tabla `utpl_productos` se puede mantener y actualizar sin tocar el flujo n8n.
- Para productos nuevos que lleguen en el futuro, basta insertar una fila en `utpl_productos`.
